DoM - document object model

  1. html codes are sent to browser
  2. browser creates a DOM model and store in its memory
  3. Html elements are the nodes of the dom tree
  4. each node has its own attribues (css) or text nodes (text)
  5. you could access any nodes to update its attributes or text, add or remove any nodes (CRUD- create, read, update, delete)
  6. the update of nodes only happened in memory, not the actual html codes itself
  7. when we found that element, we simply reference to it, rather than storing them
  8. you can also access nodeList (multi-element), e.g. getElementbyTagName("h1")
